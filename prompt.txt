Wygeneruj dane dla 6 projektów portfolio w formacie TypeScript/JavaScript object, 
zgodny z poniższym interfejsem:

interface Project {
  slug: string;
  title: string;
  category: string;        // np. "Web Design & Development", "Website Redesign"
  date: string;            // format: "DD miesiąca, YYYY" po polsku
  client: string;          // nazwa klienta
  location: string;        // miasto, kraj skrót (np. "Londyn, UK" lub "Kraków, PL")
  heroGradient: string;    // CSS gradient (radial) na hero sekcji
  coverGradient: string;   // CSS linear-gradient na mock cover image
  intro: string;           // 2-3 zdania po polsku opisujące projekt
  challenge: {
    heading: "Wyzwanie\ni rozwiązanie";
    text: string;          // 2-3 zdania o problemie klienta
    items: string[];       // 3 krótkie etapy/elementy pracy (np. "Badania użytkowników")
  };
  process: {
    heading: "Nasz\nproces";
    text: string;          // 2-3 zdania o podejściu/procesie
  };
  result: {
    heading: "Wyniki\ni efekty";
    text: string;          // 2-3 zdania o wynikach
    items: string[];       // 3 konkretne osiągnięcia z cyframi (np. "Wzrost konwersji o 45%")
  };
  img1Gradient: string;    // CSS gradient dla zdjęcia 1
  img2Gradient: string;    // CSS gradient dla zdjęcia 2
  prevSlug: string | null;
  prevTitle: string | null;
  nextSlug: string | null;
  nextTitle: string | null;
}

Wskazówki:

- Treści po polsku (pola intro, challenge.text, process.text, result.text)
- Dla projektów Yostrato (1, 2, 3, 4, 6): w polu `client` użyj angielskiej nazwy klienta, 
  w `location` podaj miasto UK. Nie musisz explicite pisać "Yostrato" w treści — 
  wystarczy że project data odzwierciedla charakter projektu zagranicznego.
- Dla P.W. Mateo: lokalizacja polska, charakter rodzimy
- heroGradient — soft radial gradient pasujący do koloru marki + neutralne tło #F5F4F8
- coverGradient — ciemny linear gradient pasujący do koloru karty z portfolio
- img1Gradient i img2Gradient — warianty coverGradient (ciemniejsze/jaśniejsze)
- Dla prevSlug/nextSlug — użyj kolejności: fabulatte → fitbox → greenwich → 
  jennings-storage → pw-mateo → padel-king → (wraca do fabulatte)
- Wyniki i liczby mają być realistyczne i wiarygodne, nie przesadzone
- Każdy opis intro musi podkreślać co KONKRETNIE zostało zbudowane (np. "sklep WooCommerce 
  z integracją płatności Stripe", "responsywna strona z systemem rezerwacji online")

Wygeneruj tablicę `const data: Project[]` z wszystkimi 6 projektami, 
gotową do wklejenia bezpośrednio do pliku [slug].astro.
